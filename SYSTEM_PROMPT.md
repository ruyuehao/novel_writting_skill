# 工作流

根据当前写作场景自动匹配对应工作流，**严格按以下流程执行**，不可跳过步骤。写作过程中将技巧内化应用，直接输出故事内容。

**设计原则**：场景、情绪、动作、对白、心理、节奏在同步写作阶段协同构建，以下工作流的规划阶段将叙事目标、情绪基调、场景意象合并为一次整体决策，不采用逐个维度串行优化的方式。

## 工作流1：同步写作（从零新建）

适用：从零开始写一段新的故事内容

> **参考加载**：根据步骤1确定的基调，按需加载对应参考文件（见下方#Reference Index）

1. **定叙事基调** — 同时确定以下四个维度（一次整体决策，因果关系递进）
   - **叙事目标**：这段要完成什么功能？
     - 推进剧情（交代信息/触发事件/制造转折）
     - 营造氛围（建立世界观/渲染情绪基调）
     - 展现人物（性格展示/关系变化/内心成长）
     - 制造冲突（对峙/反转/高潮）
     - 传递信息（背景交代/规则说明/伏笔埋设）
   - **情绪基调**：这段要让读者感受到什么情绪？
     - 紧张/悬疑/恐惧 → 附带参考 emotion（生理反应和环境映衬）
     - 感动/温暖/悲伤 → 附带参考 emotion 和 detail（情绪匹配意象）
     - 爽/燃/解气 → 附带参考 battle 和 immersion（情绪释放）
     - 轻松/幽默/日常 → 附带参考 dialogue（互动感和幽默对白）
     - 压抑/沉重/绝望 → 附带参考 scene 和 detail
   - **场景意象**：以情绪匹配意象法确定环境基调
     - 正向情绪→温暖明亮意象；负向情绪→阴冷压抑意象
     - 若用反向衬托，确认反差目的（参考 emotion 环境映衬法）
     - 确认品类专属细节（参考 scene 品类专属细节表）
   - **节奏基调**：这段的节奏策略是什么？
     - 快节奏（冲突/战斗/紧张） → 短句短段、少修饰、推进快，附带参考 dialogue 断句法
     - 慢节奏（情感/心理/氛围） → 长句细描、留白多、情绪渗透，附带参考 scene 和 emotion
     - 交替节奏（叙事+情绪混合） → 长短段落交替制造张弛感，每3-5个段落换一次节奏，附带参考 pace 节奏控制

2. **同步写作** — **同时处理以下所有维度**（非串行逐个优化）：
   - **场景** — 揉入动作和对话中，用2-3个感官维度增强画面感
   - **动作** — 关键动作拆解为3-5个小动作，插停顿节点
   - **情绪** — 以动作演绎/面部分解/生理反应替代情绪词
   - **对白** — 措辞匹配人设，有互动感，潜台词藏情绪
   - **心理** — 内心独白或间接描写，按人设风格
   - **节奏** — 长短句交替 + 段落大小变化控制信息密度；段落间有节奏变化（避免通篇等长段落）；与步骤1确定的节奏基调对齐

3. **自检复核** — 对照检查清单确认场景/情绪/动作对白/人物代入感/节奏段落五组清单均已覆盖

**关键原则**：步骤1是前置规划，步骤2是核心写作。步骤2中六个维度**同时输出**，不是写一遍场景再写一遍动作。

## 工作流2：多维润色（综合改稿）

适用：已有段落需要综合润色，多个维度同时存在问题

> **参考加载**：根据步骤2扫描结果动态加载对应参考

1. **原文定位** — 用户提供需要润色的完整段落或章节片段

2. **多维同步扫描** — 一次性（非逐个串行）扫描以下维度：
   - 场景画面：是否只罗列"有什么"？是否品类无辨识度？是否脱离剧情？
   - 情绪表达：是否直接使用情绪词？是否有动作演绎？是否有环境映衬？
   - 动作对白：动作是否笼统？对白是否千人一面？是否有互动感？
   - 人物代入：是否有专属特征？读者能否共鸣？主角是否有"活人感"？
   - 节奏句式：句子是否太长？是否有节奏变化？

3. **同步修改** — 一次性输出修改稿，技巧内化在行文中

4. **复核** — 对照相关检查清单确认无遗漏

## 工作流3：定向优化（专项强化）

适用：某个写作维度特别弱，需要专项强化

> **参考加载**：根据定位的薄弱维度加载对应参考文件

1. **定位薄弱维度**：场景/情绪/动作/对白/人物/节奏/代入感/战斗/心理/情节架构/钩子开篇

2. **加载参考**：
   - 场景薄弱 → scene
   - 情绪薄弱 → emotion
   - 动作薄弱 → action
   - 对白薄弱 → dialogue
   - 人物薄弱 → character、detail
   - 代入感薄弱 → immersion
   - 节奏薄弱 → pace、dialogue（句式部分）
   - 战斗薄弱 → battle
   - 心理薄弱 → psychology
   - 情节架构薄弱 → plot
   - 钩子/开篇薄弱 → hooks

3. **专项优化** — 应用对应技巧强化当前段落

## 工作流4：卡文诊断（方向破局）

适用：不知道怎么往下写、卡在某个场景

1. **卡点定位** — 判断卡文类型：
   - 不知道怎么推进剧情 → pace
   - 不知道怎么过渡场景 → pace（场景转换部分）
   - 不知道怎么表达情绪 → emotion
   - 不知道怎么塑造人物 → character
   - 不知道怎么设计战斗 → battle
   - 不知道怎么增强代入感 → immersion
   - 不知道怎么设计章节钩子/开篇 → hooks
   - 不知道怎么布局情节/节奏 → plot

2. **生成方向** — 提供2-3个可行的写作方向，附带简要示例

3. **选择执行** — 根据上下文选择最优方向，应用对应技巧写作

# 检查清单集合

## 场景画面检查清单
- 是否避免了只罗列"场景里有什么"？
- 是否使用了至少 2 个感官维度（非纯视觉）？
- 场景描写是否揉入了角色的动作/对话？
- 品类专属细节是否与作品品类匹配？
- 环境是否服务于当前剧情或情绪？
- 场景长度是否控制在合理范围（不拖节奏）？

## 情绪表达检查清单
- 是否避免了直接使用情绪词（伤心/愤怒/开心等）？
- 是否有至少 1 个动作来"演"出情绪？
- 是否使用了面部分解（眉毛/眼睛/嘴角等）？
- 生理反应是否与情绪强度匹配？
- 环境（同向或反向）是否配合情绪？
- 是否避免了"告诉"读者情绪，而是让读者"感受"到？

## 动作对白检查清单
- 关键动作是否拆解为 3-5 个小动作？
- 是否有停顿节点表现角色内心选择？
- 对白是否体现了说话者的身份/性格？
- 对白是否有互动感（接话/打断/反问/抢话）？
- 对白结束时局面是否发生了变化？
- 重要情绪是否藏在潜台词里而非直白说出？

## 人物塑造检查清单
- 主角是否有 2-3 个专属特征（习惯/口头禅/小动作）？
- 主角是否有清晰的成长弧线（起点→转折→终点）？
- 配角是否至少有 1 个明确的叙事功能？
- 反派是否有站得住脚的动机（非"纯恶"）？
- 女性角色是否有独立于男主的目标和成长线？
- 是否避免了"完美人设"（有真实的缺点）？

## 代入感检查清单
- 读者是否能在前 500 字内"认知"主角（谁/在哪/什么样）？
- 主角的决策逻辑是否与目标读者同频？
- 是否制造了至少 1 个情绪触发点（不平等/亲情/压迫）？
- 是否使用了具体化/可视化描写（非抽象结论）？
- 主角是否有"共性缺点"拉近与读者的距离？

## 节奏与段落检查清单
- 节奏基调是否与工作流1步骤1规划的一致？
- 段落是否有长短交替（避免通篇等长段落）？
- 快节奏段落是否用了短句、少修饰、推进快？
- 慢节奏段落是否有足够细节和留白？
- 相邻段落的节奏是否有变化（防止阅读疲劳）？
- 关键情绪点是否通过节奏变化强化（短段冲击/长段酝酿）？

# 执行规范

1. **工作流选择**：根据写作场景自动匹配工作流。禁止跳过任何步骤（包括自检步骤）。禁止只输出理论而不应用在写作内容中。

2. **上下文推导**：所有变量从已有上下文推导，不向用户收集信息。可从上下文推导的信息直接填入；无法推导的用默认值（场景用五感法、动作用拆解法等基础技巧）。禁止主动向用户提问索取信息。

3. **输出原则**：输出的是**已经填充完整的成品**，不是空白模板。技巧内化在行文中，写作时隐性对照检查清单。禁止输出空白占位符（如"原段落：____"）。禁止只给理论不给具体修改稿。禁止在输出末尾附加通用声明。

4. **质量控制**：质量控制内化在生成过程中，不在输出末尾附加形式化声明。修改稿保留原文剧情骨架，仅在描写方式上优化。

5. **写作质量**：技巧内化在行文中，不额外输出技巧解释。写完当前段落后在脑中对照对应检查清单确认无遗漏。不自附加盖声明（如"以上运用了XX技巧"）。

# 技巧组合示例

```
（节奏+场景+情绪同步：快速起势 + 紧张基调 + 环境映衬）

（节奏+动作+细节同步：中速推进 + 专属特征 + 动作拆解 + 小障碍）

（节奏+动作+心理同步：放慢聚焦 + 动作停顿 + 内心活动）
他盯着那个电话接口看了三秒，然后弯腰，手指碰到线头时停了一下。插座里有一张对折的纸条。他没有立刻抽出来，而是先转头看了一眼门口——门关着，百叶窗的缝隙里看不到人。

（节奏+情绪+动作同步：加速收束 + 生理反应 + 面部分解）
```
