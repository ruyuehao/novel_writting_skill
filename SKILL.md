# 网文写作技巧指南

## 使用指南（写作场景驱动）

### 触发条件

当 AI 正在以下写作场景中时，自动调用本 Skill 加载对应参考，无需用户主动请求：

| 写作场景 | 匹配工作流 |
|---------|-----------|
| 从零开始写一个新段落/场景 | 工作流1：同步写作 |
| 已有段落需要综合润色 | 工作流2：多维润色 |
| 某个写作维度特别弱需要专项强化 | 工作流3：定向优化 |
| 卡文/不知道怎么写/需要方向建议 | 工作流4：卡文诊断 |

### 核心工作流

AI 写作时根据当前写作场景自动匹配对应工作流，**严格按以下流程执行**，不可跳过步骤。写作过程中将技巧内化应用，直接输出故事内容。

**设计原则**：场景、情绪、动作、对白、心理、节奏在同步写作阶段协同构建，以下工作流的规划阶段将叙事目标、情绪基调、场景意象合并为一次整体决策，不采用逐个维度串行优化的方式。

#### 工作流1：同步写作（从零新建）

适用：从零开始写一段新的故事内容

> **参考加载**：根据步骤1确定的基调，按需加载对应参考文件

1. **定叙事基调** — 同时确定以下四个维度（一次整体决策，因果关系递进）
   - **叙事目标**：这段要完成什么功能？
     - 推进剧情（交代信息/触发事件/制造转折）
     - 营造氛围（建立世界观/渲染情绪基调）
     - 展现人物（性格展示/关系变化/内心成长）
     - 制造冲突（对峙/反转/高潮）
     - 传递信息（背景交代/规则说明/伏笔埋设）
   - **情绪基调**：这段要让读者感受到什么情绪？
     - 紧张/悬疑/恐惧 → 附带参考 references/04-emotion-expression.md 生理反应和环境映衬
     - 感动/温暖/悲伤 → 附带参考 references/04-emotion-expression.md 和 references/03-detail-description.md 情绪匹配意象
     - 爽/燃/解气 → 附带参考 references/09-battle-scenes.md 和 references/08-immersion.md 情绪释放
     - 轻松/幽默/日常 → 附带参考 references/05-sentence-dialogue.md 互动感和幽默对白
     - 压抑/沉重/绝望 → 附带参考 references/01-scene-atmosphere.md 和 references/03-detail-description.md
   - **场景意象**：以情绪匹配意象法确定环境基调
     - 正向情绪→温暖明亮意象；负向情绪→阴冷压抑意象
     - 若用反向衬托，确认反差目的（参考 references/04-emotion-expression.md 环境映衬法）
     - 确认品类专属细节（参考 references/01-scene-atmosphere.md 品类专属细节表）
   - **节奏基调**：这段的节奏策略是什么？
     - 快节奏（冲突/战斗/紧张） → 短句短段、少修饰、推进快，附带参考 references/05-sentence-dialogue.md 断句法
     - 慢节奏（情感/心理/氛围） → 长句细描、留白多、情绪渗透，附带参考 references/01-scene-atmosphere.md 和 references/04-emotion-expression.md
      - 交替节奏（叙事+情绪混合） → 长短段落交替制造张弛感，每3-5个段落换一次节奏，附带参考 references/07-plot-pacing.md 节奏控制

2. **同步写作** — **同时处理以下所有维度**（非串行逐个优化）：
   - **场景** — 揉入动作和对话中，用2-3个感官维度增强画面感
   - **动作** — 关键动作拆解为3-5个小动作，插停顿节点
   - **情绪** — 以动作演绎/面部分解/生理反应替代情绪词
   - **对白** — 措辞匹配人设，有互动感，潜台词藏情绪
   - **心理** — 内心独白或间接描写，按人设风格
   - **节奏** — 长短句交替 + 段落大小变化控制信息密度；段落间有节奏变化（避免通篇等长段落）；与步骤1确定的节奏基调对齐

3. **自检复核** — 对照检查清单确认场景/情绪/动作对白/人物代入感/节奏段落五组清单均已覆盖

**关键原则**：步骤1是前置规划，步骤2是核心写作。步骤2中六个维度**同时输出**，不是写一遍场景再写一遍动作。

#### 工作流2：多维润色（综合改稿）

适用：已有段落需要综合润色，多个维度同时存在问题

> **参考加载**：根据步骤2扫描结果动态加载对应参考
> 
> | 扫描维度 | 检测点 | 对应参考 |
> |---------|--------|---------|
> | 场景画面 | 是否罗列"有什么"、品类无辨识度、脱离剧情 | 【加载参考: references/01-scene-atmosphere.md】 |
> | 情绪表达 | 是否直接使用情绪词、缺乏动作演绎/环境映衬 | 【加载参考: references/04-emotion-expression.md】 |
> | 动作对白 | 动作笼统、对白千人一面、缺乏互动感 | 【加载参考: references/02-action-description.md】【加载参考: references/05-sentence-dialogue.md】（对白部分） |
> | 人物代入 | 缺乏专属特征、读者无法共鸣、主角无"活人感" | 【加载参考: references/06-character-creation.md】【加载参考: references/03-detail-description.md】【加载参考: references/08-immersion.md】 |
> | 节奏句式 | 句子过长、缺乏节奏变化 | 【加载参考: references/07-plot-pacing.md】【加载参考: references/05-sentence-dialogue.md】（句式部分） |

1. **原文定位** — 用户提供需要润色的完整段落或章节片段

2. **多维同步扫描** — 一次性（非逐个串行）扫描以下维度：
   - 场景画面：是否只罗列"有什么"？是否品类无辨识度？是否脱离剧情？
   - 情绪表达：是否直接使用情绪词？是否有动作演绎？是否有环境映衬？
   - 动作对白：动作是否笼统？对白是否千人一面？是否有互动感？
   - 人物代入：是否有专属特征？读者能否共鸣？主角是否有"活人感"？
   - 节奏句式：句子是否太长？是否有节奏变化？

3. **同步修改** — 一次性输出修改稿，技巧内化在行文中

4. **复核** — 对照相关检查清单确认无遗漏

## 综合应用建议

### 规划阶段的维度依赖关系

在定叙事基调阶段（步骤1），以下维度的信息存在前置依赖关系——确定前一项才能有效确定后一项。但进入同步写作阶段（步骤2）后，所有维度**同时输出**，不按此顺序串行写作：

1. **先定情绪** — 这段要传递什么情绪给读者（紧张/感动/爽/悲伤）
2. **选场景意象** — 用情绪匹配意象法确定环境基调
3. **设计动作** — 用动作拆解法和次级动作法写角色行动
4. **填充细节** — 用专属特征和停顿节点让角色活起来

**对白、心理、节奏**不在此依赖链中——它们贯穿所有段落，在写作阶段按需同步输出。

### 技巧组合示例

```text
（节奏+场景+情绪同步：快速起势 + 紧张基调 + 环境映衬）
大殿里的烛火忽明忽暗地跳动着。

（节奏+动作+细节同步：中速推进 + 专属特征 + 动作拆解 + 小障碍）
林秋把案上的卷宗摞整齐——这是他的习惯——才伸手去拿传讯玉简。灵力连催两次都没有回应，才发现玉简上的禁制不知何时被人封住了。

（节奏+动作+心理同步：放慢聚焦 + 动作停顿 + 内心活动）
他盯着那枚暗淡的玉简看了三秒，放下玉简弯下腰，手指触到案台底下时顿了一下。案台下贴着一张对折的符纸。他没有立刻撕下来，而是先转头看了一眼门口——门关着，窗纸缝隙里看不到人影。

（节奏+情绪+动作同步：加速收束 + 生理反应 + 面部分解）
他感觉后背的冷汗正往下淌。把符纸撕下来展开，看到上面的字时，他的瞳孔猛地收缩了一下。嘴角抿成一条线，他把符纸揉进手心，攥紧。
```

## 检查清单集合

### 场景画面检查清单

- [ ] 是否避免了只罗列"场景里有什么"？
- [ ] 是否使用了至少 2 个感官维度（非纯视觉）？
- [ ] 场景描写是否揉入了角色的动作/对话？
- [ ] 品类专属细节是否与作品品类匹配？
- [ ] 环境是否服务于当前剧情或情绪？
- [ ] 场景长度是否控制在合理范围（不拖节奏）？

### 情绪表达检查清单

- [ ] 是否避免了直接使用情绪词（伤心/愤怒/开心等）？
- [ ] 是否有至少 1 个动作来"演"出情绪？
- [ ] 是否使用了面部分解（眉毛/眼睛/嘴角等）？
- [ ] 生理反应是否与情绪强度匹配？
- [ ] 环境（同向或反向）是否配合情绪？
- [ ] 是否避免了"告诉"读者情绪，而是让读者"感受"到？

### 动作对白检查清单

- [ ] 关键动作是否拆解为 3-5 个小动作？
- [ ] 是否有停顿节点表现角色内心选择？
- [ ] 对白是否体现了说话者的身份/性格？
- [ ] 对白是否有互动感（接话/打断/反问/抢话）？
- [ ] 对白结束时局面是否发生了变化？
- [ ] 重要情绪是否藏在潜台词里而非直白说出？

### 人物塑造检查清单

- [ ] 主角是否有 2-3 个专属特征（习惯/口头禅/小动作）？
- [ ] 主角是否有清晰的成长弧线（起点→转折→终点）？
- [ ] 配角是否至少有 1 个明确的叙事功能？
- [ ] 反派是否有站得住脚的动机（非"纯恶"）？
- [ ] 女性角色是否有独立于男主的目标和成长线？
- [ ] 是否避免了"完美人设"（有真实的缺点）？

### 代入感检查清单

- [ ] 读者是否能在前 500 字内"认知"主角（谁/在哪/什么样）？
- [ ] 主角的决策逻辑是否与目标读者同频？
- [ ] 是否制造了至少 1 个情绪触发点（不平等/亲情/压迫）？
- [ ] 是否使用了具体化/可视化描写（非抽象结论）？
- [ ] 主角是否有"共性缺点"拉近与读者的距离？

### 节奏与段落检查清单

- [ ] 节奏基调是否与步骤1规划的一致？
- [ ] 段落是否有长短交替（避免通篇等长段落）？
- [ ] 快节奏段落是否用了短句、少修饰、推进快？
- [ ] 慢节奏段落是否有足够细节和留白？
- [ ] 相邻段落的节奏是否有变化（防止阅读疲劳）？
- [ ] 关键情绪点是否通过节奏变化强化（短段冲击/长段酝酿）？

## LLM执行规范

> 本章节面向 AI 模型，规定调用本 Skill 时的执行标准，确保输出可预测、可操作、不遗漏。

### 1. 工作流选择规则

写作场景与工作流匹配关系见上方「使用指南」表格，AI 根据当前场景自动匹配。

**禁止行为**：
- 禁止跳过工作流中的任何步骤（包括自检步骤）
- 禁止只输出理论而不应用在写作内容中

### 2. 上下文推导规则

本 Skill 在被 AI 写作时自动调用，不向用户收集信息。所有变量从已有上下文推导：

| 变量 | 推导来源 | 处理策略 |
|------|---------|-------------|
| 问题类型 | 当前正在创作的段落类型 | 根据触发条件自动匹配对应工作流 |
| 作品品类 | 已有上下文中的作品设定 | 从对话上下文提取，无法推导则跟随已有设定 |
| 当前原文 | AI 刚刚写出的文字 | 直接使用 AI 输出的上一段或当前段 |
| 写作目标 | 当前段落在叙事中的功能 | 从上下文的剧情位置推导 |

**变量填充规则**：
1. 可从上下文推导的信息 → 直接填入，不额外标注
2. 无法推导的信息 → 使用默认值（场景用五感法、动作用拆解法等基础技巧）
3. 禁止主动向用户提问索取信息

### 3. 模板填充规则

**核心原则**：AI 输出的是**已经填充完整的成品**，不是空白模板。

**操作步骤**：
1. 定位对应工作流后，加载对应参考文件获取技巧要点
2. 在写作过程中将技巧内化应用，不输出模板格式（写作内容即为成品）
3. 写作时隐性对照检查清单确保质量

**禁止行为**：
- 禁止输出空白占位符（如"原段落：____"）
- 禁止只给理论不给具体修改稿
- 禁止在输出末尾附加通用声明（如"以上技巧可根据实际情况调整"）

### 4. 质量控制原则

质量控制应内化在生成过程中，而非在输出末尾附加形式化声明。

**生成时约束（隐性执行，不输出推理过程）**：
- 填充模板前，先对照对应检查清单确认关键要素已覆盖
- 修改稿必须保留原文剧情骨架，仅在描写方式上优化
- 所有推导假设和待用户决策点，直接在主体内容的相应字段括号内标注"（推导）"或"待用户确认"

**各工作流对应约束清单**：
- 工作流1（同步写作）：生成前确认场景/情绪/动作对白/人物代入感/节奏段落五组检查清单均已满足
- 工作流2（多维润色）：复核时确认所有相关检查清单已满足
- 工作流3（定向优化）：确认对应维度的检查清单已满足
- 工作流4（卡文诊断）：确认生成的方案方向有参考支撑

### 5. 写作质量要求

**写作时隐性约束**：
- 技巧应内化在行文中，不额外输出技巧解释（除非折叠标注）
- 写完当前段落后，在脑中对照对应检查清单确认无遗漏
- 不自附加盖声明（如"以上运用了XX技巧"）
